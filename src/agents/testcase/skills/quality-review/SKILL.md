# 质量自检与评审 Skill

## 激活场景
- 每个模块用例输出完毕后
- 全部用例生成完成后
- 用户要求评审测试用例质量
- 需要输出测试报告

---

## 10 项快速自检清单

每个模块用例输出完毕后，执行以下检查：

```
□ 1. 所有功能点均有用例覆盖
     检查：功能清单 vs 用例清单，确保一一对应

□ 2. 每个用例预期结果具体可验证（无模糊描述）
     检查：无"正常"、"成功"、"正确"等模糊词

□ 3. 每个用例提供了具体测试数据
     检查：数据字段非空，且有具体值

□ 4. P0用例数量 ≥ 3条
     检查：核心功能至少有3条阻塞级用例

□ 5. 包含安全相关用例（如有用户输入）
     检查：SQL注入、XSS、越权等场景覆盖

□ 6. 包含至少1条异常场景用例
     检查：每个功能点有正常+异常覆盖

□ 7. 用例编号无重复、格式规范
     检查：TC-XXX-XXX-NNN 格式，无重复

□ 8. 前置条件均可独立准备
     检查：无依赖其他用例执行结果的情况

□ 9. 测试步骤步数合理（5-15步）
     检查：步骤过多则拆分，过少则合并

□ 10. 预期结果涵盖UI层与数据层验证
     检查：不只有界面验证，还有数据验证
```

## 质量维度评估

### 1. 覆盖度评估

| 评估项 | 权重 | 评估标准 | 评分 |
|--------|------|----------|------|
| 功能覆盖 | 30% | 功能点覆盖率 ≥ 95% | 0-100 |
| 场景覆盖 | 25% | 正常+异常+边界场景 | 0-100 |
| 优先级分布 | 20% | P0:P1:P2:P3 合理 | 0-100 |
| 类型覆盖 | 15% | 功能/接口/安全/性能 | 0-100 |
| 数据覆盖 | 10% | 有效/无效/边界数据 | 0-100 |

**覆盖度计算公式**：
```
覆盖度得分 = Σ(各项得分 × 权重)
等级：
- 优秀：≥90分
- 良好：80-89分
- 合格：70-79分
- 需改进：<70分
```

### 2. 可执行性评估

| 检查项 | 合格标准 | 评分权重 |
|--------|----------|----------|
| 步骤清晰度 | 操作描述可执行 | 30% |
| 预期结果可验证 | 有明确验证标准 | 30% |
| 数据可准备 | 测试数据可构造 | 20% |
| 环境可搭建 | 前置条件可实现 | 20% |

### 3. 冗余度检查

**识别冗余用例**：
- 多条用例验证同一检查点
- 用例之间存在包含关系
- 过于相似的测试步骤

**处理策略**：
- 合并相似用例
- 删除重复覆盖
- 标记为"可选执行"

## 常见问题与修复建议

### 问题1：预期结果模糊
```
❌ 问题：预期结果包含"正常显示"
✅ 修复：改为"页面显示商品列表，每页20条，共5页"
```

### 问题2：缺少异常场景
```
❌ 问题：只有正常流程用例
✅ 修复：补充网络异常、超时、权限不足等场景
```

### 问题3：测试数据缺失
```
❌ 问题：测试数据列为空
✅ 修复：填充具体值，如 username: "test001"
```

### 问题4：用例冗余
```
❌ 问题：3条用例验证同一登录成功场景
✅ 修复：合并为1条，其他改为不同场景
```

### 问题5：编号不连续
```
❌ 问题：编号为 TC-001, TC-003, TC-005
✅ 修复：重新编号为 TC-001, TC-002, TC-003
```
